# 搭把手订单状态机设计

## 订单状态定义

| 状态码 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 已取消 | 订单被取消 |
| 1 | 待支付 | 等待买家支付积分 |
| 2 | 已支付 | 积分已冻结到担保池 |
| 3 | 服务中 | 卖家已开始提供服务 |
| 4 | 待确认 | 服务完成，等待买家确认 |
| 5 | 已完成 | 订单完成，积分结算 |
| 6 | 已退款 | 订单退款 |
| 7 | 争议中 | 订单存在争议 |

## 状态流转规则

```
待支付(1) → 已支付(2) → 服务中(3) → 待确认(4) → 已完成(5)
    ↓           ↓           ↓           ↓
  已取消(0)   已退款(6)   已退款(6)   争议中(7)
                ↓           ↓           ↓
            已取消(0)    已取消(0)    已完成(5)/已退款(6)
```

## 关键业务规则

### 1. 支付环节
- 买家支付积分，积分从买家余额扣除
- 积分冻结到担保池（系统账户）
- 生成动态核销码（6位数字，30分钟有效）

### 2. 服务确认
- 线下服务：卖家扫码核销
- 线上服务：买家点击确认
- 核销码超时未使用自动熔断

### 3. 超时熔断机制
- 待支付超时（15分钟）→ 自动取消
- 核销码超时（30分钟）→ 自动退款
- 服务确认超时（7天）→ 自动完成

### 4. 退改扣分规则
- 买家取消：已支付后取消扣10%积分
- 卖家取消：接单后取消扣20%积分
- 争议处理：由系统仲裁
