# 前端API调用目录

本目录只维护前端到后端的接口调用函数，禁止在页面、组件或组合式函数中直接硬编码后端URL。

## 文件分工

- `auth.ts`：登录、注册、刷新token、退出登录。
- `user.ts`：用户资料、校园认证、信任分等个人中心接口。
- `skill.ts`：技能分类、技能标签、用户技能。
- `shelf.ts`：技能货架发布、管理、查询。
- `demand.ts`：需求发布、管理、看板查询。
- `order.ts`：订单创建、支付、状态流转、核销。
- `point.ts`：积分账户、流水、签到、担保池。
- `message.ts`：通知、聊天会话、聊天消息。
- `credit.ts`：评价、违规、申诉。
- `stat.ts`：个人统计和平台统计。
- `admin.ts`：管理后台聚合接口。

## 维护规则

- 请求参数名必须与后端DTO一致，分页统一使用`pageNum`、`pageSize`。
- 响应类型必须引用`frontend/src/types/api.ts`或模块化类型文件。
- 新增或修改接口时，必须同步更新`docs/api/`中的模块文档。
